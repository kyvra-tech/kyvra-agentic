# Design: Kyvra ↔ TrendPost Integration

Generated by /plan-ceo-review on 2026-03-20. Status: ACTIVE.

## Problem Statement

Kyvra (agentic-kyvra) generates content intelligence: scored news stories, threads, briefs, newsletters, scripts — via Telegram and a REST API.

TrendPost (content-creator-tool) is a publishing platform: users create content, schedule it, and publish to Twitter, LinkedIn, Threads.

Today they are **disconnected**. Users manually copy content ideas from Telegram to TrendPost. This friction kills the workflow.

## Goal

Connect them: Kyvra feeds TrendPost with daily intelligence. TrendPost users discover top stories, generate content, and publish — without leaving TrendPost.

## Architecture

```
agentic-kyvra (Python/FastAPI, port 8000)
  GET /update?module=X     ──────────▶ kyvra-client.service.ts (TrendPost)
  GET /thread?module=X     ──────────▶   - fetchTopStories()
  GET /brief?module=X      ──────────▶   - generateContent()
  GET /status?module=X     ──────────▶ kyvra.routes.ts (/api/kyvra/*)
  POST /signal/performance ◀────────── analytics signal (on post publish)
  8AM: webhook push        ──────────▶ webhooks.routes.ts → kyvra_stories table

TrendPost (Node.js/Express/PostgreSQL)
  DB tables (new):
    kyvra_stories:          global inbox (module, title, url, signal_label, pushed_at)
    user_kyvra_preferences: user_id, kyvra_module (default='tech')
  UI (new):
    /dashboard/kyvra        Kyvra Feed Tab
    KyvraFeed.tsx           Story cards with signal badges
    KyvraStoryCard.tsx      Generate Thread | Brief | Newsletter buttons
```

## Data Model

### kyvra_stories (PostgreSQL)
```sql
CREATE TABLE kyvra_stories (
  id               SERIAL PRIMARY KEY,
  module           TEXT NOT NULL,           -- 'tech'|'crypto'|'vietnam'|'indie'
  title            TEXT NOT NULL,
  url              TEXT NOT NULL,
  confidence_score INTEGER NOT NULL,
  signal_label     TEXT NOT NULL,           -- 'VIRAL'|'RISING'|'STEADY'|'SIGNAL'
  summary          TEXT,
  pushed_at        TIMESTAMPTZ NOT NULL DEFAULT NOW(),
  UNIQUE (url, module, DATE(pushed_at))     -- idempotent push
);
CREATE INDEX idx_kyvra_stories_module_time ON kyvra_stories (module, pushed_at DESC);
```

### user_kyvra_preferences (PostgreSQL)
```sql
CREATE TABLE user_kyvra_preferences (
  user_id         UUID PRIMARY KEY REFERENCES users(id) ON DELETE CASCADE,
  kyvra_module    TEXT NOT NULL DEFAULT 'tech',
  updated_at      TIMESTAMPTZ NOT NULL DEFAULT NOW()
);
```

## Key Decisions

- **Global inbox model**: Kyvra push creates shared `kyvra_stories` rows, not per-user drafts. All users with matching module preference see the same feed. Scales to any number of users.
- **Backend proxy**: TrendPost backend proxies all Kyvra API calls. KYVRA_API_KEY is never exposed to the browser.
- **15-min cache**: TrendPost caches Kyvra /update responses 15 min per module. Without this, every feed tab load triggers a full Kyvra pipeline run.
- **HMAC auth on webhooks**: Both webhook push (Kyvra→TrendPost) and performance signals (TrendPost→Kyvra) use HMAC-SHA256 with a shared secret.
- **Feature flag**: `KYVRA_INTEGRATION_ENABLED=true/false` in TrendPost env. Allows deployment without enabling.

```

## User Flow

```
User opens TrendPost → /dashboard/kyvra tab
  → sees 7 story cards (VIRAL/RISING/STEADY labels)
  → clicks "Generate Thread" on top story
  → 30-45s wait (Kyvra LLM runs)
  → thread opens in content editor, pre-filled with user's voice
  → schedules to Twitter → published
  → TrendPost emits performance signal to Kyvra
  → Kyvra adjusts source authority score (+delta)
```

## Deployment Sequence

1. Deploy Kyvra with TRENDPOST_WEBHOOK_URL set → verify push logs
2. Deploy TrendPost DB migrations (2 new tables)
3. Deploy TrendPost backend with KYVRA_INTEGRATION_ENABLED=false
4. Set KYVRA_INTEGRATION_ENABLED=true → smoke test
5. Deploy TrendPost frontend with Kyvra Feed Tab

## UI Design Specification (from /plan-design-review 2026-03-20)

### Story Card Anatomy

```
┌─────────────────────────────────────────────────────────────┐
│ [SIGNAL BADGE]  ·  [Source name]  ·  [Time ago]     [↗ N]  │  ← metadata (text-xs gray-500)
│                                                              │
│  Story title — text-base font-semibold text-gray-900        │  ← PRIMARY (largest text)
│  (max 2 lines, line-clamp-2)                                │
│                                                              │

├─────────────────────────────────────────────────────────────┤
│  [🧵 Thread]   [⚡ Brief]   [📰 Newsletter]   [↗ View]     │  ← actions (always visible)

### Page Header Layout

```
┌────────────────────────────────────────────────────────┐
│  Kyvra Feed  (font-display Cal Sans, text-2xl)         │
│  [Tech] [Crypto] [Vietnam] [Indie]  ← pill tabs        │

```

Pill tabs: active = `bg-primary-500 text-white`, inactive = `bg-gray-100 text-gray-700`, `rounded-full px-3 py-1 text-sm`

### Generation Progress Sequence (30-45s wait UX)

```
0-2s:  Button text → "Fetching story..."
2-10s: Button text → "Analyzing content..."
10-25s: Button text → "Writing [format]..."
25-45s: Button text → "Almost done..."
45s+:  Button text → "This is taking longer than usual..."
60s:   Timeout → Toast error "Timed out. Try again in a minute."
```

Button state during generation: `variant="primary" disabled + Spinner` (from existing components)

### Post-Generate Flow

After generation completes:
1. Save content as TrendPost draft via `/api/content/save` (or pass as query param)
2. Navigate to `/create?draft=[id]` with content pre-filled
3. Toast: "✅ Thread ready — edit and schedule."
4. User edits and schedules as normal

### Interaction State Coverage

| Feature | Loading | Empty | Error | Success |
|---------|---------|-------|-------|---------|
| Feed tab | 7× CardSkeleton shimmer | "Daily stories arrive at 8AM 🌅" + SparklesIcon | "Kyvra feed unavailable ↻ Try again" | 7 story cards + freshness badge |
| Generate button | Disabled + progress text sequence | — | Toast "Timed out" + re-enable | Auto-navigate to editor |
| Module switch | Feed skeleton while loading | Same empty state | Same error state | New stories for module |
| Morning push | — | — | Logged only | Orange dot on Kyvra nav tab |

### Voice Profile Indicator

When voice is set (per user from `/setvoice`):
```
🎤 Your voice active · "Casual, punchy, data-driven"
```
When not set:
```
+ Set your voice for personalized output  [links to /setvoice docs or settings]
```

### Responsive Layout

**Desktop/Tablet (≥768px):** `max-w-2xl mx-auto`, cards stacked vertically, 4 action buttons inline
**Mobile (<768px):** Horizontal swipe card deck (CSS scroll-snap), one card at a time, dot pagination indicator (● ○ ○ ○ ○ ○ ○), action buttons 2×2 grid, module pills scrollable horizontally

### Accessibility

- Action buttons: `aria-label="Generate Thread for: [story title]"`
- Signal badge: `aria-label="Signal strength: VIRAL"`
- Story link: `target="_blank" rel="noopener noreferrer"`
- Loading: `aria-live="polite"` on feed container
- Progress text: `aria-live="polite" aria-atomic="true"`
- Touch targets: `min-h-[44px] min-w-[44px]` on all interactive elements
- Focus visible: `ring-2 ring-primary-500`
